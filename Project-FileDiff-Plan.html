Project Concept · Architecture · Runbook

Desktop App FileDiff

A spartan yet elegant side-by-side diff tool for the desktop. Pick two files and see them aligned line for line with color-coded insertions, deletions, and changes — plus word-level highlights inside every modified line. A second mode compares two folders recursively and drills into any changed file. Built on Electron with zero build tooling.

01The concept

FileDiff is a local, single-window diff viewer — the kind of thing you reach for when you want to see exactly what changed between two files without opening a heavyweight IDE or a web service. It models the experience of tools like Beyond Compare and Araxis Merge, distilled to the one thing they do best: a clean, aligned, color-coded side-by-side view.

What it does

  • Side-by-side file diff with line-number gutters and rows that stay aligned — filler rows hold the place of inserted/removed lines so both sides line up.
  • Color-coded changes: inserted removed changed — kept muted, never loud.
  • Word-level intraline highlights mark the exact words that changed within a modified line.
  • Folder compare walks two directory trees and shows added / removed / modified / identical per file; click a modified file to diff it.
  • Synchronized scrolling, jump-to-next/previous change, swap sides, reload, and ignore-whitespace.

Why these choices

  • Electron gives native file dialogs, drag-and-drop, and direct filesystem access in one cross-platform shell.
  • Vanilla JS, no bundler — the truest expression of "spartan." It boots instantly and has the fewest moving parts to break.
  • jsdiff is a battle-tested diff engine; diffLines builds the row structure and diffWordsWithSpace drives the intraline marks.
  • Plain CSS + design tokens keep the visual system consistent with the project's warm-paper aesthetic.
Scope, v1: "simple diff display to start." Syntax-color tokenization is intentionally deferred — combining per-token coloring with word-level diff marks reliably is fiddly, and the elegance here comes from typography, spacing, and the diff palette. It's the natural Phase 2 enhancement.

The renderer never touches Node or the filesystem directly. With contextIsolation: true and nodeIntegration: false, the only bridge is a small, explicit API surface exposed by preload.js — openFile, openFolder, readFile, compareFolders, join. Every one of those is an ipcRenderer.invoke answered by a handler in the main process. Diffing itself runs entirely in the renderer, on text the main process already read and validated.

File reads are guarded: an 8 MB size cap and a binary sniff (a NUL byte in the first 8 KB) keep the renderer responsive and surface a friendly message instead of hanging on a huge or binary file.

03The diff algorithm

The heart of the app lives in renderer/diff-view.js. It turns two strings into two aligned columns of rows in five steps:

  1. Line diff. Diff.diffLines(left, right, { ignoreWhitespace }) returns ordered chunks, each flagged added, removed, or unchanged.
  2. Align into row pairs. Unchanged → same row on both sides. A pure removal → text on the left, a hatched filler on the right (and vice-versa for an insertion) so line numbers never drift.
  3. Detect changes. A removed chunk immediately followed by an added chunk is treated as a changed block and paired line-for-line.
  4. Intraline marks. For each changed pair, Diff.diffWordsWithSpace(old, new) wraps only the differing words in <mark> — removed words tint the left, added words tint the right.
  5. Render & index. Rows render into both panes; the first row of each maximal change run is recorded so Next/Prev can jump between hunks. A footer tallies inserted / removed / changed / unchanged.

04Folder compare

The second mode (lib/folder-compare.js + renderer/folder-view.js) walks both directory trees and keys every entry by its relative path:

StatusMeaning
addedPresent only in the right folder.
removedPresent only in the left folder.
modifiedIn both, but different — detected by size first, then a SHA-1 content hash to rule out false positives.
identicalIn both, byte-for-byte equal.

Aggregate counts render in the header, and a click on any modified row reads that file pair and hands it to the side-by-side diff view. .git, node_modules, and .DS_Store are skipped while walking.

06The application

Title bar & mode switch

  • A slim warm header with the FileDiff wordmark and a pill switch between Files and Folders modes.

Toolbar

  • Left and Right slots, each with a Browse button and a drag-and-drop target that shows the chosen file/folder name.
  • Swap sides, Reload, an Ignore-WS toggle, and ↑ / ↓ change navigation.

Diff panes & stat bar

  • Two synced columns with line-number gutters; row backgrounds encode status and <mark>s encode word changes.
  • A footer reads N inserted · N removed · N changed · N unchanged with matching color chips.
  • Empty state invites you to pick two files; identical files render neutral with all-unchanged stats.

08Commands — run & package

First run

# from the project root (file-diff/)
npm install
npm start            # launches the Electron app

Verify (headless screenshots)

# renders the real renderer offscreen and writes PNGs to samples/
./node_modules/.bin/electron scripts/screenshot.js
./node_modules/.bin/electron scripts/screenshot-folder.js

Package a .app / .dmg (optional, Phase 2)

# add electron-builder, then:
npm run dist         # output: dist/
Roadmap. Syntax highlighting (per-line, composed under the diff marks), inline (unified) diff toggle, recent-pairs memory, configurable ignore globs for folder compare, and a proper packaged build with an .icns icon.
